# Flat Hunter v1.0
Автоматизированная микросервисная система непрерывного мониторинга сайтов аренда недвижимости
## Цель проекта
Проект Flat Hunter - это парсер объявлений недвижимости (Яндекс), который сохраняет новые квартиры в БД и отправляет уведомления в Telegram.
## Архитектура системы
Проект построен на базе микросервисной архитектуры и разворачивается в изолированных Docker-контейнерах:
* **Parser Service (`/parser`)** — скачивает HTML-страницы, парсит объявления и передает данные.
* **Bot Service (`/bot`)** — взаимодействует с пользователем через Telegram и отправляет уведомления.
* **Database (`PostgreSQL`)** — хранит историю объявлений, чтобы избегать дубликатов.
## Стек
Python 3.11, PostgreSQL (в Docker), requests, beautifulsoup4, psycopg2-binary, aiogram (для бота).
## Инфраструктура
Проект полностью контейнеризован (docker-compose.yml). Запущено два контейнера: db (Postgres) и bot (Python-скрипт).
## Проблема 
Сейчас парсер работает как монолит в бесконечном цикле while True с time.sleep(300) внутри файла main.py. Это блокирует поток, мешает масштабированию и добавлению новых парсеров (например, для других сайтов).


# Flat Hunter v2.0

## Цель спринта
Уйти от блокирующего цикла while True и перевести архитектуру проекта на асинхронные очереди задач.
Внедрить брокер сообщений Redis и инструмент управления задачами Celery. 

## Архитектурные изменения
### Система должна быть разделена на следующие роли (в рамках Docker Compose):

- База данных (PostgreSQL): Хранит обработанные квартиры (остается без изменений).

- Брокер сообщений (Redis): Новая сущность. Хранит очередь задач.

- Celery Beat (Планировщик): Помещает задачу "спарсить Яндекс" в очередь (Redis) каждые 5 минут.

- Celery Worker (Работник): Непрерывно слушает очередь Redis. Как только видит задачу — берет её, запускает парсер, пишет в БД и дергает бота для отправки сообщения.
